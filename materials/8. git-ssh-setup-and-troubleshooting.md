## Конспект: Налаштування SSH для GitHub та Усунення Помилок

SSH (Secure Shell) — це безпечний протокол для підключення до GitHub без використання HTTPS-адреси та паролів.

### 1\. Генерація та Додавання SSH-Ключа

| Крок                            | Команда                                                                          | Пояснення                                                                                    |
| :------------------------------ | :------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------- |
| **Генерація ключа**             | `ssh-keygen -t ed25519 -C "ваш_email@example.com"`                               | Створює пару ключів (приватний і публічний). Рекомендований сучасний алгоритм — **ed25519**. |
| **Збереження**                  | Натисніть **`Enter`** для збереження у стандартному місці (`~/.ssh/id_ed25519`). | _Опціонально:_ Можна додати парольну фразу (passphrase) для додаткової безпеки.              |
| **Запуск агента**               | `eval "$(ssh-agent -s)"`                                                         | Переконайтеся, що SSH-агент запущений.                                                       |
| **Додавання ключа**             | `ssh-add ~/.ssh/id_ed25519`                                                      | Додає ваш приватний ключ до агента, щоб не вводити парольну фразу при кожному пуші.          |
| **Копіювання публічного ключа** | `cat ~/.ssh/id_ed25519.pub`                                                      | Виводить публічний ключ (який починається з `ssh-ed25519...`) для копіювання.                |

### 2\. Додавання Ключа до GitHub

1.  **Перейдіть** до **GitHub Settings** → **SSH and GPG keys**.
2.  Натисніть **"New SSH Key"**.
3.  У полі **"Title"** введіть опис (наприклад, "Мій ноутбук").
4.  У полі **"Key"** вставте **скопійований публічний ключ**.
5.  Натисніть **"Add SSH Key"**.

### 3\. Тестування З'єднання та Налаштування Remote

| Дія                      | Команда                                              | Пояснення                                                                                                             |
| :----------------------- | :--------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------- |
| **Тестування з'єднання** | `ssh -T git@github.com`                              | При першому підключенні введіть `yes`. У разі успіху ви побачите повідомлення "You've successfully authenticated...". |
| **Перевірка Remote URL** | `git remote -v`                                      | Перевіряє, чи підключений репозиторій.                                                                                |
| **Додавання Remote SSH** | `git remote add origin git@github.com:user/repo.git` | Встановлює адресу віддаленого репозиторію, використовуючи **безпечний SSH-протокол**.                                 |

### 4\. Усунення Помилок `git push`

| Проблема                                                    | Команда                                             | Пояснення                                                                                                                                                                                |
| :---------------------------------------------------------- | :-------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Відхилено через віддалені зміни** (Updates were rejected) | `git pull --rebase origin main`                     | Завантажує віддалені зміни, застосовує ваші локальні коміти поверх них. **Обов'язково** робіть це перед повторним `git push`.                                                            |
| **src refspec main does not match any**                     | Спочатку виконайте `git commit -m "Initial commit"` | Помилка виникає, коли ви намагаєтеся відправити гілку, в якій немає жодного коміту.                                                                                                      |
| **Примусове перезаписування** ( Небезпечно\!)               | `git push --force origin main`                      | **Перезаписує** історію на віддаленому сервері вашою локальною версією. Видаляє будь-які зміни, зроблені іншими користувачами. Використовувати тільки у крайніх, усвідомлених випадках\! |

