## Повернення до Попередніх Версій (Restore, Reset)

Цей розділ охоплює команди для скасування змін у робочій директорії, індексі (Staging Area) та самій історії комітів.

### 1\. Повернення до Попередніх Версій Файлів

Команди `git restore` та `git checkout` використовуються для роботи з окремими файлами.

| Дія                                               | Команда                | Пояснення                                                                                                     |
| :------------------------------------------------ | :--------------------- | :------------------------------------------------------------------------------------------------------------ |
| **Видалити файл**                                 | `rm file.txt`          | Видаляє файл лише з **робочої директорії** (файл стає `deleted`).                                             |
| **Відновити файл**                                | `git restore file.txt` | Скасовує будь-які локальні зміни у файлі, відновлюючи його стан з **останнього коміту** або **Staging Area**. |
| **Видалити файл з індексу та Робочої Директорії** | `git rm file.txt`      | Видаляє файл з обох місць та додає цю дію до Staging Area.                                                    |

#### Відновлення файлу з історії

1.  **Перевірка історії:**

    ```bash
    git log --oneline
    ```

    _(Знайдіть хеш коміту, з якого потрібно відновити файл, наприклад, `abc123`.)_

2.  **Перегляд змін у коміті:**

    ```bash
    git show abc123
    ```

    _(Дозволяє переглянути, як виглядав файл у конкретному коміті.)_

3.  **Відновлення файлу з конкретного коміту:**

    ```bash
    git restore --source=abc123 file.txt
    ```

    _(Файл `file.txt` буде відновлено до версії коміту `abc123`. **Увага:** Ця версія з'явиться у вашій робочій директорії як змінена, але не зафіксована.)_

4.  **Фіксація відновленого стану:**

    ```bash
    git add file.txt
    git commit -m "Restore file.txt from abc123"
    ```

### 2\. Повернення до Попередніх Версій Проєкту (`git reset`)

Команда `git reset` дозволяє перемістити вказівник історії (`HEAD`) на старіший коміт, ефективно **переписуючи історію**.

| Команда                       | Призначення                                                         | Наслідки                                                                                                                 |
| :---------------------------- | :------------------------------------------------------------------ | :----------------------------------------------------------------------------------------------------------------------- |
| **`git reset --hard HEAD`**   | Скасувати всі **локальні** зміни (у робочій директорії та індексі). | **Небезпечно\!** Всі зміни будуть **безповоротно втрачені**.                                                             |
| **`git reset --hard abc123`** | Переміщує `HEAD` на коміт `abc123`.                                 | **Безповоротно видаляє** всі коміти **після** `abc123` з локальної історії та **стирає** всі зміни з робочої директорії. |
| **`git reset --soft abc123`** | Переміщує `HEAD` на коміт `abc123`.                                 | **Зберігає** всі зміни у **Staging Area**. Ви можете відразу створити новий коміт, який охоплює всі ці зміни.            |
