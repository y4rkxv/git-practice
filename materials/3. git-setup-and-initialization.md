### Налаштування та Ініціалізація Git

### 1\. Перевірка та Ідентифікація Git

Перш ніж почати працювати з Git, потрібно підтвердити, що програма встановлена та налаштована для ідентифікації ваших змін.

| Дія                        | Команда                                          | Призначення                                                                                                 |
| :------------------------- | :----------------------------------------------- | :---------------------------------------------------------------------------------------------------------- |
| **Перевірка встановлення** | `git --version`                                  | Виводить поточну версію Git, підтверджуючи, що програма доступна в терміналі.                               |
| **Встановлення імені**     | `git config --global user.name "Ваше Ім'я"`      | Визначає ваше ім'я для всіх майбутніх комітів.                                                              |
| **Встановлення Email**     | `git config --global user.email "ваш@email.com"` | Визначає ваш email. Ці дві змінні **обов'язкові** для виконання будь-яких комітів.                          |
| **Перевірка налаштувань**  | `git config --list`                              | Виводить повний список усіх глобальних і локальних налаштувань Git (включно з `user.name` та `user.email`). |

### 2\. Створення Локального Репозиторію

Репозиторій — це місце, де Git зберігає всю історію та метадані проєкту.

- **`git init --initial-branch=main`**

  - Ця команда **ініціалізує** Git у поточній папці (перетворює звичайну папку на Git-репозиторій).
  - Створює **приховану папку `.git/`**, в якій зберігаються всі метадані, історія та конфігурація.
  - Одразу створює головну гілку під назвою **`main`**.

- **Прихована папка `.git/`**

  - **Увага:** Ми **ніколи не змінюємо** та не видаляємо нічого в цій папці вручну. Це "мозок" Git.

### 3\. Перевірка Робочого Стану

| Дія                        | Команда      | Призначення                                                                                                                |
| :------------------------- | :----------- | :------------------------------------------------------------------------------------------------------------------------- |
| **Перевірка статусу Git**  | `git status` | Показує поточний стан робочої директорії та Staging Area: які файли змінені, які готові до коміту, а які не відстежуються. |
| **Перевірка вмісту папки** | `ls -a`      | Команда терміналу, яка виводить список усіх файлів у поточній папці, включно з прихованими файлами, такими як `.git/`.     |
| **Довідка Git**            | `git --help` | Виводить загальну довідку Git у терміналі (корисно, якщо ви забули основну                                                 |
| команду).                  |
