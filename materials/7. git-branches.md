## Конспект: Робота з Гілками (Branches)

Гілки дозволяють розробляти нові функції, виправляти помилки або експериментувати, не впливаючи на стабільний код у головній гілці (`main`).

### 1\. Навігація та Створення Гілок

| Дія                                           | Команда                           | Пояснення                                                                                       |
| :-------------------------------------------- | :-------------------------------- | :---------------------------------------------------------------------------------------------- |
| **Перевірка наявних гілок**                   | `git branch`                      | Виводить список усіх локальних гілок. Поточна гілка позначається зірочкою (`*`).                |
| **Створення нової гілки**                     | `git branch feature`              | Створює нову гілку з назвою `feature`, яка є точною копією поточної гілки.                      |
| **Перемикання на гілку**                      | `git checkout feature`            | Переміщує вказівник `HEAD` на гілку `feature`. Тепер всі коміти будуть відбуватися в цій гілці. |
| **Створення та перемикання (швидкий спосіб)** | `git checkout -b new_branch_name` | Одночасно створює нову гілку з назвою `new_branch_name` **та** одразу перемикається на неї.     |

### 2\. Робота з Гілкою та Фіксація Змін

Коміти, зроблені в новій гілці, не впливають на гілку `main`.

1.  **Внесення змін** (додайте новий файл або змініть існуючий).
2.  **Додайте та зафіксуйте зміни:**
    ```bash
    git add .
    git commit -m "Оновлення файлів"
    ```
3.  **Перевірте історію:** `git log` покаже, що новий коміт існує лише у вашій поточній гілці (наприклад, `feature`).

### 3\. Об'єднання Гілок (Merge)

Після завершення роботи, зміни з робочої гілки переносяться в головну (`main`).

1.  **Перемкніться на цільову гілку:**
    ```bash
    git checkout main
    ```
2.  **Перевірте відмінності** (опціонально):
    ```bash
    git diff new_branch_name
    ```
    _(Показує, які саме зміни будуть внесені в `main`.)_
3.  **Об'єднайте гілки:**
    ```bash
    git merge new_branch_name
    ```
    _(Git бере всі коміти з `new_branch_name` і додає їх у `main`, створюючи новий коміт злиття, якщо це необхідно.)_

### 4\. Видалення Гілки

Коли гілка була успішно злита, її варто видалити, щоб уникнути плутанини.

| Команда                             | Призначення                                                                                                   | Коли використовувати                                            |
| :---------------------------------- | :------------------------------------------------------------------------------------------------------------ | :-------------------------------------------------------------- |
| **`git branch -d new_branch_name`** | **Безпечне видалення.** Видаляє гілку лише у тому випадку, якщо вона була **повністю злита** з гілкою `main`. | Завжди використовуйте це першим\!                               |
| **`git branch -D new_branch_name`** | **Примусове видалення.** Видаляє гілку, навіть якщо вона не була злита (тобто, ви **втратите** ті коміти).    | Використовуйте тільки, якщо ви впевнені, що коміти не потрібні. |
