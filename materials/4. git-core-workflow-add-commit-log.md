### Основний Робочий Процес Git (Add, Commit, Log)

### 1\. Створення Файлу та Перший Коміт

- **Перевірка статусу (до роботи):**

  ```bash
  git status
  ```

  _(Показує, що "рабоча директорія чиста" або є невідстежувані файли.)_

- **Створення нового файлу:**

  ```bash
  touch file_name.txt
  ```

  _(Файл з'являється у робочій директорії, але ще **не відстежується** Git'ом.)_

- **Додавання файлу в Staging Area (Зона індексації):**

  ```bash
  git add file_name.txt
  ```

  _(Файл стає **кандидатом** на коміт.)_

- **Виконання коміту (фіксація):**

  ```bash
  git commit -m "Add file"
  ```

  _(Створює **знімок** проєкту. Файл переміщується з Staging Area у локальний репозиторій.)_

### 2\. Зміна Коду та Новий Коміт

Щоб зафіксувати зміни в існуючому файлі, цикл повторюється:

1.  **Змініть файл** (за допомогою VS Code або термінального редактора, наприклад `nano file_name.txt`).
2.  **Перевірте статус** (`git status` покаже файл як `modified`).
3.  **Додайте змінений файл** у Staging Area:
    ```bash
    git add file_name.txt
    ```
4.  **Виконайте коміт** з новим описом:
    ```bash
    git commit -m "Update file"
    ```

### 3\. Аналіз Історії Комітів

Використовуйте команду `git log` для перегляду всіх зафіксованих змін.

- **Повний перегляд історії:**

  ```bash
  git log
  ```

  _(Показує повні дані: хеш, автора, дату, повідомлення.)_

- **Компактний перегляд:**

  ```bash
  git log --oneline
  ```

  _(Показує історію в один рядок: скорочений хеш + повідомлення.)_

- **Перегляд змін у комітах (Diff):**

  ```bash
  git log -p
  ```

  _(Показує не тільки повідомлення, але й усі **відмінності** (що було додано/видалено) у кожному коміті. Натисніть **`q`** для виходу.)_

### 4\. Перегляд Відмінностей та Коміт через Редактор

- **Перевірка відмінностей у файлі (до додавання в Staging):**

  ```bash
  git diff
  ```

  _(Показує, що було змінено у файлі відносно останнього коміту.)_

- **Виконання коміту з повідомленням через редактор:**

  ```bash
  git commit
  ```

  _(Якщо ви не використовуєте прапорець `-m`, Git відкриє встановлений редактор (VS Code або Notepad) для введення детального повідомлення.)_

### 5\. Налаштування Редактора За Замовчуванням

Ці команди встановлюють, який редактор буде відкриватися, коли ви виконуєте команду `git commit` без прапорця `-m`.

- **Встановлення Notepad (Windows):**
  ```bash
  git config --global core.editor "notepad"
  ```
- **Встановлення VS Code (рекомендовано):**
  ```bash
  git config --global core.editor "code --wait"
  ```
  _(`--wait` гарантує, що Git чекатиме, поки ви закриєте VS Code, перш ніж завершити коміт.)_
