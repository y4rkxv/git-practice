### Фундамент Роботи в Терміналі (Bash/MinGW64)

| **Key Questions / Keywords**    | **Notes / Explanations**                                                                                                                                                                                                          |
| :------------------------------ | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Що таке Bash/Термінал?**      | **Bash (MinGW64)** — це командний інтерфейс (CLI), який дозволяє взаємодіяти з операційною системою за допомогою тексту, а не миші. Це основа для роботи з Git, оскільки Git не має графічного інтерфейсу (GUI) за замовчуванням. |
| **ls: Перегляд вмісту**         | Команда **`ls`** (list) показує, які файли та папки знаходяться в поточній директорії. Це аналог відкриття папки у Провіднику Windows. <br>**Приклад:** `ls *.csv` покаже тільки файли з розширенням CSV.                         |
| **pwd: Де я зараз?**            | Команда **`pwd`** (print working directory) відображає повний шлях до поточної папки. Це критично, щоб не загубитися перед виконанням важливих скриптів чи команд Git.                                                            |
| **cd: Перехід між папками**     | Команда **`cd`** (change directory) використовується для навігації. <br>**Логіка:** `cd <назва_папки>` переходить у цю папку; `cd ..` піднімає на один рівень вище.                                                               |
| **mkdir & touch: Створення**    | **`mkdir`** (make directory) створює нову папку. <br>**`touch`** створює новий порожній файл. <br>**Логіка:** Це основні інструменти для організації структури проєкту перед початком роботи.                                     |
| **cp & rm: Управління файлами** | **`cp`** (copy) копіює файл/папку, дозволяючи зберегти резервну копію оригіналу: `cp source destination`. <br>**`rm`** (remove) видаляє файл. **Обережно:** `rm -r <папка>` видаляє папку разом з усім вмістом без кошика.        |
| **grep: Пошук у тексті**        | Команда **`grep`** сканує текстові файли і виводить рядки, що містять конкретний шаблон або слово. <br>**Застосування:** Дуже корисна для пошуку помилок (`grep "ERROR" logs.txt`) або змінних у коді.                            |
| **clear**                       | **`clear`** очищує екран терміналу від попередніх команд і виводу, забезпечуючи чистоту робочого простору.                                                                                                                        |

### Summary (Резюме своїми словами)

Команди Bash (терміналу) — це фундаментальна операційна система для роботи з файлами, папками та текстом усередині Git Bash. Вони дозволяють створювати, переміщувати, переглядати, копіювати та видаляти об'єкти без використання миші. **`ls`**, **`cd`** та **`pwd`** є базою для навігації. Розуміння цих команд є необхідною умовою для ефективного використання Git, оскільки саме в цих папках і файлах Git відстежує зміни.

---

### Практичні ментальні моделі

1.  **Термінал — це робот-помічник:** Ти не махаєш мишкою, а даєш роботу-помічнику чіткі текстові інструкції (команди).
2.  **`ls` — це "Подивитися":** Як відкрити папку.
3.  **`cd` — це "Перейти":** Як подвійний клік на папці.
4.  **Файлова система — це Дерево:** Корінь (`/`) — це основа, а папки — це гілки. Команда `pwd` показує, на якій гілці ти сидиш.
5.  **Git живе у папці:** Команди Git (наприклад, `git status`) працюють тільки в тому каталозі, де ти зараз перебуваєш. Тому перед Git завжди треба перевіряти `pwd`.
6.  **`grep` — це "Пошук по тексту":** Як натиснути `Ctrl+F` у великому файлі, але набагато швидше і потужніше.

### Типові помилки початківців

1.  **Плутанина між `rm` та `rm -r`:** Використовувати `rm <папка>` для видалення папки (не спрацює). Потрібно `rm -r <папка>` (рекурсивне видалення), що є небезпечним.
2.  **Відсутність шляху:** Виконувати команду `cd <назва_папки>` тоді, коли цієї папки немає в поточному каталозі. Потрібно або перейти туди, де вона є, або вказати повний шлях.
3.  **Регістр:** У терміналі (Bash/Linux) назви файлів і папок чутливі до регістру. `cd project` та `cd Project` — це різні команди.
4.  **Спроба видалити папку без прапорця `-r`:** `rm myfolder` не видалить папку. Потрібно `rm -r myfolder`.
5.  **Забуття, де ти є:** Запускати скрипт або команду Git, не перевіривши `pwd`, і дивуватися, чому команда працює у неправильній папці.
