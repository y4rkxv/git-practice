## Розширення Основ Git (Staging, Amend, Clean)

### 1\. Повідомлення та Редагування Комітів

| Дія                               | Команда                             | Пояснення                                                                                                              |
| :-------------------------------- | :---------------------------------- | :--------------------------------------------------------------------------------------------------------------------- |
| **Створення коміту**              | `git commit -m "Add file test.txt"` | Стандартний спосіб створення **нового** знімка (коміту).                                                               |
| **Редагування останнього коміту** | `git commit --amend --no-edit`      | **Замінює** останній коміт новим. Використовується, щоб додати забутий файл до останньої фіксації або виправити вміст. |
| **Перевірка змін**                | `git log`                           | Переконайтеся, що старий коміт замінений новим.                                                                        |

### 2\. Розширена Робота зі Staging Area

- **Додавання багатьох файлів:**

  - Ви можете вказати всі файли, які хочете додати до Staging Area, в одному рядку:

  <!-- end list -->

  ```bash
  git add file1.txt file2.txt existing_file.txt
  ```

- **Перегляд змін:**
  | Команда | Що порівнюється | Коли використовувати |
  | :--- | :--- | :--- |
  | **`git diff`** | Робоча директорія **vs** Staging Area | Показати, що ви **ще не додали** командою `git add`. |
  | **`git diff --cached`** | Staging Area **vs** Останній коміт | Показати, що ви **додали** командою `git add`, але ще не зафіксували. |

- **Відміна індексації (Unstaging):**

  - Ця команда видаляє файл зі Staging Area, повертаючи його до Робочої директорії (як `modified`).

  <!-- end list -->

  ```bash
  git restore --staged file1.txt
  ```

  _(Файл залишається зміненим, але більше не є кандидатом на коміт.)_

### 3\. Очищення Робочої Директорії (Cleaning)

- **`git clean`** — видаляє файли, які **не відстежуються** Git'ом (наприклад, тимчасові файли компілятора, які ви не додали до `.gitignore`).

| Команда            | Призначення                                                                               | Увага                                              |
| :----------------- | :---------------------------------------------------------------------------------------- | :------------------------------------------------- |
| **`git clean -n`** | **"Dry Run"** (Перевірка). Показує, які файли **будуть видалені**, але нічого не видаляє. | Завжди запускайте це спочатку\!                    |
| **`git clean -f`** | **Force** (Примусово). Видаляє невідстежувані файли.                                      | **Обережно:** Видалені файли неможливо відновити\! |
